Use this section when you work on the engine repository. For application
development, start with [Develop with Byte](/docs/use).

## Test your changes

Run the full test suite:

```bash
cargo test
```

Some rendering tests need a local GPU. For a quick logic-only pass, skip tests
whose names contain `render` or `graphics`:

```bash
cargo test -- --skip render --skip graphics
```

## Measure code coverage

Install `cargo-llvm-cov`:

```bash
cargo install cargo-llvm-cov
```

Then generate an LCOV report:

```bash
cargo llvm-cov --lcov --output-path coverage/lcov.info
```

To view the report in Visual Studio Code, install the `markis.code-coverage`
extension.

## Write documentation

Use [ISO 24495-1 plain-language principles](https://www.iso.org/standard/78907.html)
for Rust comments and API documentation. Keep useful links between types, traits,
functions, methods, and modules even when a strict plain-language rewrite would
remove them. Developers use these links to navigate the API.

Use the [Microsoft Writing Style Guide](https://learn.microsoft.com/style-guide/)
for pages in `docs`. Address the reader directly, lead with their goal, use
familiar words, and make the next action clear.
